# 04 · 调试能力设计

> 目标：把"出错时一脸懵"变成"出错时有友好提示 + 一键拿到诊断信息 + 直接交给 AI 修"。
>
> 与 [02 日志系统](./02-日志系统设计.md) 联动，构成"发现问题 → 定位问题 → 交给 AI 修复"完整链路。

---

## 1. 调试能力三层

```
① 终端用户层   友好错误页 error.vue —— 不吓到人，提供"复制诊断"按钮
② 制作者层     应用内调试面板（开发浮层）—— 实时看日志/路由/状态/请求
③ 开发工具层   Nuxt DevTools —— 组件树、状态、路由、性能，开箱即用
```

非程序人员主要用 ① 和 ②，AI 拿到 ① 导出的诊断信息后定位修复。

---

## 2. 友好错误页（`error.vue`）

Nuxt 中根目录的 `error.vue` 会捕获致命错误。要做到：**不暴露吓人的堆栈给终端用户，但给"复制诊断信息"按钮**。

```vue
<!-- error.vue -->
<script setup lang="ts">
import { getLogBuffer } from '~/composables/useLogger'

const props = defineProps<{ error: { statusCode: number; message: string; stack?: string } }>()
const log = useLogger('error-page')
const copied = ref(false)

log.error('页面致命错误', { statusCode: props.error.statusCode, message: props.error.message })

// 一键打包诊断信息（错误 + 最近日志 + 环境）
function buildDiagnostics() {
  return JSON.stringify({
    error: {
      statusCode: props.error.statusCode,
      message: props.error.message,
      stack: props.error.stack,
    },
    route: useRoute().fullPath,
    time: new Date().toISOString(),
    userAgent: navigator.userAgent,
    recentLogs: getLogBuffer().slice(-50),
  }, null, 2)
}

async function copyDiagnostics() {
  await navigator.clipboard.writeText(buildDiagnostics())
  copied.value = true
  setTimeout(() => (copied.value = false), 2000)
}
</script>

<template>
  <div class="min-h-screen flex items-center justify-center p-6">
    <UCard class="max-w-lg w-full text-center">
      <UIcon name="i-lucide-triangle-alert" class="text-5xl text-amber-500 mb-4" />
      <h1 class="text-xl font-semibold mb-2">页面出了点问题</h1>
      <p class="text-gray-500 mb-6">
        别担心，这不是你的错。点击下方按钮复制诊断信息，发给 AI 助手即可帮你修复。
      </p>

      <div class="flex gap-3 justify-center">
        <UButton icon="i-lucide-clipboard-copy" @click="copyDiagnostics">
          {{ copied ? '已复制 ✓' : '复制诊断信息' }}
        </UButton>
        <UButton color="neutral" variant="soft" icon="i-lucide-home" @click="clearError({ redirect: '/' })">
          返回首页
        </UButton>
      </div>

      <!-- 仅开发态展示技术细节 -->
      <details v-if="$nuxt?.isHydrating || true" class="mt-6 text-left text-xs text-gray-400">
        <summary class="cursor-pointer">技术细节（给开发者看）</summary>
        <pre class="mt-2 whitespace-pre-wrap">{{ error.message }}</pre>
      </details>
    </UCard>
  </div>
</template>
```

> 关键：**"复制诊断信息"= 错误 + 最近 50 条日志 + 路由/UA/时间**。非程序人员粘贴给 AI，AI 几乎能直接定位。这是整套方案的"闭环出口"。

## 3. 应用内调试面板（开发浮层 `DevPanel.vue`）

仅在开发态（`import.meta.dev`）渲染的右下角浮层，实时显示日志流，无需打开浏览器控制台。

```vue
<!-- components/DevPanel.vue -->
<script setup lang="ts">
import { getLogBuffer } from '~/composables/useLogger'
const open = ref(false)
const logs = ref<ReturnType<typeof getLogBuffer>>([])
let timer: any

onMounted(() => {
  if (!import.meta.dev) return
  timer = setInterval(() => { logs.value = getLogBuffer().slice(-100).reverse() }, 500)
})
onUnmounted(() => clearInterval(timer))

const route = useRoute()
const levelColor = (l: string) =>
  l === 'error' ? 'text-red-500' : l === 'warn' ? 'text-amber-500' : 'text-gray-500'
</script>

<template>
  <ClientOnly>
    <div v-if="import.meta.dev" class="fixed bottom-4 right-4 z-[9999]">
      <UButton icon="i-lucide-bug" color="neutral" @click="open = !open" />
      <div v-if="open" class="mt-2 w-[420px] h-[320px] bg-black/90 text-white rounded-lg p-3 overflow-auto text-xs font-mono">
        <div class="mb-2 text-gray-400">路由：{{ route.fullPath }}</div>
        <div v-for="(l, i) in logs" :key="i" class="border-b border-white/10 py-1">
          <span :class="levelColor(l.level)">[{{ l.level }}]</span>
          <span class="text-indigo-400"> {{ l.module }}</span> · {{ l.message }}
          <span v-if="l.context" class="text-gray-500"> {{ JSON.stringify(l.context) }}</span>
        </div>
      </div>
    </div>
  </ClientOnly>
</template>
```

在 `app.vue` 引入 `<DevPanel />` 即可（生产自动不渲染）。

> 价值：制作者不用懂"打开 F12 控制台"，直接在页面右下角看到日志流。

## 4. Nuxt DevTools（开箱即用）

`nuxt.config.ts` 中已默认开启：

```typescript
export default defineNuxtConfig({
  devtools: { enabled: true },
})
```

提供：组件树、页面/路由、状态、模块、性能时间线、虚拟文件。对进阶排查很有用，AI 也常引用其概念。

## 5. `npm run doctor` —— 一键体检

封装一条命令，收集环境与配置信息，输出可粘贴给 AI 的体检报告：

```json
{ "scripts": { "doctor": "node scripts/doctor.mjs" } }
```

```javascript
// scripts/doctor.mjs
import { execSync } from 'node:child_process'
import { readFileSync, existsSync } from 'node:fs'

const out = {}
out.node = process.version
out.npm = execSync('npm -v').toString().trim()
out.os = `${process.platform} ${process.arch}`
if (existsSync('package.json')) {
  const pkg = JSON.parse(readFileSync('package.json', 'utf8'))
  out.deps = pkg.dependencies
  out.devDeps = pkg.devDependencies
}
out.hasAgentsMd = existsSync('AGENTS.md')
out.hasErrorVue = existsSync('error.vue')
out.hasTests = existsSync('tests')
console.log('===== 项目体检报告（可复制给 AI）=====')
console.log(JSON.stringify(out, null, 2))
```

## 6. 错误处理约定（写进 AGENTS.md）

```markdown
## 错误与调试规范
- 致命错误交给 error.vue 兜底，不要在页面里写裸 try/catch 吞掉错误
- 可恢复错误用 log.warn 并给用户友好提示（UToast/UAlert）
- 不可恢复错误用 log.error，让 error.vue 显示"复制诊断信息"
- 开发态保留 <DevPanel />，生产自动隐藏
- 排查问题先看 DevPanel 日志流 / error.vue 诊断信息
```

```
  error.vue 友好提示 ──► 点击"复制诊断信息"

        │                       ▼
        │                 粘贴给 AI 助手
        ▼                       │
  useLogger 已记录 ◄────────────┘
  DevPanel 实时可见        AI 定位 → 改代码 → npm run check
        ▲                                   │
        └───────────── 绿灯，问题闭环 ◄──────┘
```
