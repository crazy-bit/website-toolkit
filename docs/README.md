# AI 建站工具 · 方案文档

> 面向**非程序人员**的 AI 建站工程化脚手架方案。
> 解决"用 AI 做网站，页面一多就代码混乱、调试困难、错误频出"的问题。

## 一句话定位

> 一套**让 AI 在护栏内工作、并能自动获得反馈**的工程化脚手架（Guardrails + Feedback Loop for AI Coding）。

## 四大支柱与闭环

```
统一架构  ──►  约束 AI 怎么写（Constraint）
日志系统  ──►  出错时能看见发生了什么（Observability）
测试体系  ──►  自动验证改动有没有破坏功能（Verification）
调试能力  ──►  快速定位并修复问题（Diagnosability）
      ＋ 贯穿主线：AI 协作规范（AGENTS.md）+ 一键命令

  描述需求 → AI 写代码+测试 → 一键 check → 红灯则看日志自修复 → 绿灯
```

## 参考实现

本仓库 `maoxianjia/` 已采用完整目标技术栈（Nuxt 3 + Nuxt UI + GSAP + Lenis + Iconify），
可作为脚手架原型（P0）抽象来源。需补齐：`AGENTS.md`、`error.vue`、`useLogger`、`tests/`。

## 给非程序人员的极简使用心智

| 想做的事 | 一条命令 |
|----------|----------|
| 启动 | `npm run dev` |
| 改完检查有没有坏 | `npm run check` |
| 出问题打包给 AI | `npm run doctor` / 点错误页"复制诊断信息" |
